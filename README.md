# index-liquipedia-journal

## 简介

`index-liquipedia-journal` 是一个用于归档和发布独立 HTML 页面的仓库。  
本仓库不针对任何单一域名或具体网站，仅作为一系列静态页面的集合与版本管理用途。

## 目录说明

```
.
├── README.md          # 本说明文件
├── index.html         # 主入口页面（可选）
├── pages/             # 存放归档 HTML 页面的目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
└── assets/            # 资源文件（如 CSS、JS、图片等，如有）
    └── ...
```

- `pages/`：主要归档目录，每个独立的 HTML 页面可存放于此，便于分类与引用。
- `assets/`：存放页面所需的公共资源文件，保持页面结构清晰。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，页面之间无直接依赖关系。  
每个页面可视为一个独立的记录或展示单元，适合用于保存特定时间点的内容快照或静态展示。

归档时建议遵循以下原则：

- 每个 HTML 文件应能够独立运行，不依赖外部网络资源（除非必要）。
- 文件命名应简洁明了，便于识别。
- 如需引用资源，请将资源放置于 `assets/` 目录下，并使用相对路径引用。

## 维护说明

- 本仓库将持续收录新的 HTML 页面，但不对内容时效性作保证。
- 欢迎提交 Pull Request 添加新的归档页面，但请确保页面内容不违反 GitHub 服务条款。
- 对于过时或不再需要的页面，可提交 Issue 讨论是否移除。
- 维护者将定期检查仓库结构，保持目录整洁。
